# PROJECT VISION

## 1. Why（为什么做）

- 打造一个可快速嵌入博客的交互式 Tone Mapping 教学工程。
- 先建立可靠的 Linear 输入与诊断基线，再逐步扩展到多算法对比（ACES / AgX / Reinhard / GT7 / LPM 等）。

## 2. Vision（愿景）

- 提供“看得见、调得动、可解释”的实时图像映射学习体验。
- 保持技术实现最小但可扩展：先正确，再丰富。
- 让内容既能作为教学演示，也能直接复用于真实图形管线验证。

## 3. Product Principles（产品原则）

1. 交互优先：每个核心参数都应可实时观察结果。
2. 可信优先：算法实现必须可追溯到参考资料或原始来源。
3. 最小复杂度：避免过早引入重框架与重流程。
4. 可嵌入性：默认可通过单 JS 包接入博客页面。

## 4. Current Direction（当前方向）

- 阶段 1：完成 WebGL2 Linear Baseline（已启动）
- 阶段 1.5：补充 HDR/EXR 测试素材集（优先真实场景与可复现公开数据）
- 阶段 2：接入单一 Tone Mapping 算法作为对照
- 阶段 3：扩展为多算法切换与统一参数面板
- 阶段 4：完善教学文案、可视化说明与示例场景

## 5. Non-goals（当前非目标）

- 不做企业级项目管理流程（严格 Spec/Progress 看板）。
- 不追求一次性覆盖所有渲染后处理能力。
- 不在早期引入复杂工程编排（多仓库、多运行时）。

## 6. Decision Log（关键决策记录）

- 采用原生 WebGL2 作为最小实现路径。
- 采用 Vite + TypeScript 作为开发与构建基础。
- 文档治理采用单文件愿景模式，不使用严格规格模板。
- 测试图像策略新增约束：优先使用 HDR/EXR 文件，测试数据以公开可追溯来源为准。
- EXR 作为额外测试输入正式接入 baseline demo，支持与既有 HDR 输入统一切换验证。
